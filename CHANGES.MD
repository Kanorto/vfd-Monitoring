- Переписан вывод в `vfd_monitor.py`: формирование экрана теперь идет через фиксированные 20-символьные строки и побайтовую кодировку служебных символов дисплея, из-за чего устранено смешивание строк, мерцание и разъезд символов.
- Добавлены безопасные функции отрисовки `write_screen`/`encode_vfd_text`, блокировка доступа к COM-порту и более надежное переподключение дисплея.
- Логика отображения метрик переработана: первая строка стабильно показывает CPU/GPU/RAM, вторая — скорости диска и сети, с компактным форматированием под 2x20 VFD.
- Добавлены гибкие переключатели метрик в трее: теперь можно отдельно включать и отключать CPU/GPU/RAM, температуры, диск и сеть, а строки VFD динамически пересчитываются и центрируются под текущий набор пунктов.
- Переработан алгоритм компоновки строк: сегменты автоматически уплотняются только при нехватке места, поэтому при отключении части метрик оставшиеся элементы корректно центрируются без ручной подгонки.
- Добавлен `requirements.txt` и GitHub Actions workflow `.github/workflows/release.yml`, который на теге `v*` собирает `VFD PC Monitor.exe` через PyInstaller и сразу публикует его в GitHub Release без использования artifacts.
- Добавлена полноценная подсистема автообновления из GitHub Releases: приложение хранит текущую и ожидаемую версии в конфиге, фоном проверяет релизы, скачивает свежий `.exe`/`.py` с повторными попытками и безопасно применяет обновление через отдельный `.cmd`-обновлятор при следующем запуске или выходе.
- После установки новой версии приложение сравнивает встроенную версию с `app_version` из конфига, пишет changelog в лог, сохраняет его для последующего просмотра и показывает краткое уведомление на VFD.
- В трей добавлено меню `Обновления` с ручной проверкой, принудительной установкой скачанного обновления и просмотром changelog, а на самом VFD появились анимированные статусы проверки/загрузки обновления с прогресс-баром.
- Автозапуск Windows теперь использует корректную команду запуска для `exe` и `py`-режима, поэтому обновлённый бинарник продолжает подниматься автоматически после замены файла.

- Репозиторий обновлений по умолчанию теперь жёстко задан как `Kanorto/vfd-Monitoring`, а после нескольких подряд неудачных попыток автообновления приложение показывает alert со ссылкой на релиз для ручного скачивания.
- Поведение трея доработано: после переключения отображаемых метрик, автозапуска и скорости меню принудительно обновляется через `icon.update_menu()`, чтобы приложение не завершалось и не теряло состояние при настройке пунктов `Что показывать`.
- На стартовом экране теперь второй строкой показывается текущая версия приложения, а компоновщик основных метрик сначала пытается использовать полные префиксы `CPU`/`GPU`/`RAM` и сокращает их только при нехватке ширины VFD.
- Выбор файла обновления переработан так, чтобы автообновление в первую очередь забирало любой `.exe` из релиза GitHub, даже если рядом опубликован файл с точным совпадением имени текущего `.py`.
