# VFD Monitoring

Небольшое Windows-приложение для системного мониторинга и вывода служебной информации на двухстрочный VFD-дисплей 2×20. Проект изначально рассчитан на **ATOL PD-2800**, но подходит и для других VFD-устройств, которые определяются как USB-COM и принимают совместимую последовательность команд.

Программа работает в фоне, показывает загрузку системы, умеет отображать заметки и напоминания, живёт в системном трее, поддерживает горячие клавиши и может обновляться через GitHub Releases без ручной переустановки.

## Что умеет

### Системный мониторинг
- выводит на первую строку загрузку CPU, температуру CPU, загрузку GPU, температуру GPU и RAM;
- выводит на вторую строку скорость чтения/записи диска и входящего/исходящего сетевого трафика;
- автоматически ужимает шаблоны и отступы, если строка перестаёт помещаться в 20 символов;
- корректно считает ширину VFD-символов и специальных байтов, чтобы строки не «разъезжались» на реальном дисплее;
- умеет отключать отдельные метрики прямо из трея.

### Работа с дисплеем
- автоматически ищет подходящий COM-порт среди устройств на базе Prolific, FTDI, Posiflex, ATOL и совместимых USB-to-Serial адаптеров;
- инициализирует VFD в нужной кодировке и пишет экран как две фиксированные строки по 20 символов;
- показывает приветственное сообщение при подключении и аккуратно очищает дисплей при выходе;
- переживает потерю соединения с COM-портом и пытается переподключиться автоматически.

### Заметки и напоминания
- поддерживает заметки с индивидуальным интервалом показа;
- поддерживает три типа напоминаний: разовые по дате и времени, ежедневные и еженедельные по выбранным дням;
- показывает длинные сообщения постранично, адаптируя текст под формат 2×20;
- даёт приоритет сработавшим напоминаниям перед обычными системными метриками;
- хранит историю скрытых, выполненных и удалённых элементов;
- ограничивает число одновременно активных заметок и напоминаний через конфиг.

### Интерфейс и управление
- работает через иконку в системном трее;
- позволяет включать и выключать дисплей, менять скорость обновления и состав метрик;
- открывает отдельные окна управления заметками, напоминаниями и горячими клавишами;
- умеет быстро открыть конфиг и лог-файл;
- поддерживает глобальные горячие клавиши с переназначением из интерфейса.

### Обновления
- проверяет новые версии в GitHub Releases;
- скачивает свежий релиз в папку `updates`;
- умеет установить скачанное обновление при следующем запуске или во время выхода;
- хранит changelog последних 10 релизов и показывает его в отдельном окне;
- после нескольких неудачных попыток подсказывает ссылку на релиз для ручного обновления.

## Как это выглядит на устройстве

Базовый режим рассчитан на **две строки по 20 символов**:
- **строка 1** — CPU / GPU / RAM;
- **строка 2** — диск / сеть;
- если активна заметка или напоминание, они временно перехватывают экран;
- служебные статусы вроде проверки обновлений или успешного действия из хоткея тоже выводятся поверх основного экрана.

## Требования

### ОС
Проект ориентирован на **Windows**. В коде используются `winreg`, `pywin32`, `WMI`, глобальные hotkeys через WinAPI и Windows-логика автозапуска, поэтому на Linux и macOS приложение в текущем виде не поддерживается.

### Python
- для запуска из исходников: Python 3.11+ на Windows;
- для сборки релизного `.exe`: **строго Python 3.11**.

### Зависимости рантайма
Устанавливаются из `requirements.txt`:
- Pillow
- psutil
- py3nvml
- pyserial
- pystray
- pywin32
- WMI

### Зависимости для сборки
Устанавливаются из `requirements-build.txt`:
- все зависимости рантайма;
- `pyinstaller>=6.13,<7`.

## Быстрый старт

### 1. Клонирование
```bash
git clone https://github.com/Kanorto/vfd-Monitoring.git
cd vfd-Monitoring
```

### 2. Установка зависимостей
```bash
python -m pip install -r requirements.txt
```

### 3. Запуск
```bash
python vfd_monitor.py
```

После первого запуска рядом с программой будет создан файл `vfd_config.json`, а также лог `vfd_monitor.log`.

## Управление из трея

В главном меню доступны:
- включение и выключение дисплея;
- выбор отображаемых метрик;
- выбор скорости обновления: `0.5`, `1.0`, `5.0` секунды или свой вариант;
- блок управления горячими клавишами;
- меню заметок и напоминаний;
- меню обновлений;
- открытие конфига и логов;
- автозапуск с Windows;
- корректный выход с очисткой дисплея.

## Заметки и напоминания

### Заметки
- каждая заметка имеет текст, флаг активности и собственный интервал показа;
- одновременно можно держать включёнными не более 10 заметок по умолчанию;
- длинный текст разбивается на страницы 2×20 и показывается последовательно;
- после полного показа заметка возвращается в обычный цикл до следующего интервала.

### Напоминания
Поддерживаются три сценария:
1. **Разовое** — по конкретной дате и времени.
2. **Ежедневное** — каждый день в указанное время.
3. **Еженедельное** — в выбранные дни недели и в указанное время.

Сработавшее напоминание:
- получает приоритет над системными метриками;
- остаётся активным, пока его не скрыть или не отметить выполненным;
- попадает в историю после завершения действия.

## Настройка через `vfd_config.json`

Конфиг создаётся автоматически, но его можно править вручную. Основные поля:

| Ключ | Назначение |
|---|---|
| `show_cpu_usage`, `show_cpu_temp` | Показ загрузки и температуры CPU |
| `show_gpu_usage`, `show_gpu_temp` | Показ загрузки и температуры GPU |
| `show_ram` | Показ RAM |
| `show_disk` | Показ скорости диска |
| `show_network` | Показ сети |
| `autostart` | Автозапуск с Windows |
| `update_interval` | Частота обновления данных на дисплее |
| `github_repo` | Репозиторий для автообновлений |
| `metric_formats` | Шаблоны строк для метрик |
| `line_spacing` | Отступы между блоками в обычном и компактном режиме |
| `hotkeys_enabled` | Глобальное включение hotkeys |
| `hotkeys` | Назначения клавиш |
| `notes_reminders` | Полный блок заметок, напоминаний, лимитов и истории |

### Пример структуры
```json
{
  "show_cpu_usage": true,
  "show_cpu_temp": true,
  "show_gpu_usage": true,
  "show_gpu_temp": true,
  "show_ram": true,
  "show_disk": true,
  "show_network": true,
  "autostart": false,
  "update_interval": 1.0,
  "github_repo": "Kanorto/vfd-Monitoring",
  "metric_formats": {
    "cpu": ["CPU{usage:02d}% {temp:02d}°", "C{usage:02d}%"],
    "gpu": ["GPU{usage:02d}% {temp:02d}°", "G{usage:02d}%"],
    "ram": ["RAM{value:02d}%"],
    "disk": ["D:{read}/{write}"],
    "network": ["N:{recv}/{send}"]
  },
  "line_spacing": {
    "primary": " ",
    "secondary": " ",
    "primary_compact": "",
    "secondary_compact": ""
  },
  "hotkeys_enabled": true,
  "hotkeys": {
    "toggle_display": "Ctrl+Alt+D",
    "check_updates": "Ctrl+Alt+U",
    "show_changelog": "Ctrl+Alt+C"
  },
  "notes_reminders": {
    "settings": {
      "limits": {
        "max_active_notes": 10,
        "max_active_reminders": 50,
        "max_history_items": 200
      },
      "display": {
        "note_page_duration_seconds": 2.4,
        "reminder_page_duration_seconds": 3.2
      },
      "defaults": {
        "note_interval_seconds": 30
      }
    }
  }
}
```

### Пользовательское форматирование метрик

Для `metric_formats` доступны такие плейсхолдеры:

- `cpu`, `gpu`: `{usage}`, `{temp}`, `{full_prefix}`, `{short_prefix}`, `{degree}`;
- `ram`: `{value}`;
- `disk`: `{read}`, `{write}`;
- `network`: `{recv}`, `{send}`.

Приложение перебирает шаблоны сверху вниз и выбирает первый вариант, который влезает в строку вместе с остальными сегментами.

## Автообновления

Механика работы простая:
1. приложение проверяет GitHub Releases указанного репозитория;
2. забирает changelog и сведения о последнем релизе;
3. пытается скачать подходящий файл релиза;
4. в приоритете выбирает `.exe`, затем точное совпадение с текущим именем файла, затем запасной вариант;
5. откладывает установку до безопасного момента;
6. после обновления сохраняет историю релизов и changelog в конфиг.

По умолчанию используется репозиторий `Kanorto/vfd-Monitoring`, но его можно заменить на свой форк, если соблюдён формат релизов и в них публикуется исполняемый файл.

## Сборка `.exe`

### Важно
Релизная сборка официально поддерживается только на **Windows + Python 3.11**. Это сделано специально, чтобы избежать нестабильных бинарников и проблем вроде `Failed to load Python DLL ... python313.dll`.

### Установка зависимостей для сборки
```bash
python -m pip install -r requirements-build.txt
```

### Запуск сборки
```bash
python build_windows_release.py
```

Скрипт:
- проверит, что вы на Windows;
- проверит версию Python;
- очистит старые папки `build` и `dist`;
- соберёт один файл `VFD PC Monitor.exe` через PyInstaller.

## Если что-то не работает

### Дисплей не находится
Проверьте:
- определяется ли устройство в диспетчере устройств как COM-порт;
- не занят ли порт другой программой;
- подходит ли USB-UART чип под текущую автоидентификацию;
- принимает ли ваш дисплей ту же инициализацию и кодировку, что и PD-2800.

### Нет температуры GPU
Для NVIDIA используется `py3nvml`. Если драйвер или NVML недоступны, приложение продолжит работать, но без метрик GPU.

### Нет температуры CPU
Температура CPU берётся через WMI. На некоторых системах этот источник может быть недоступен или возвращать пустые значения.

### Горячие клавиши не срабатывают
Проверьте:
- включены ли hotkeys в настройках;
- не занята ли комбинация другой программой;
- есть ли права на регистрацию глобальных hotkeys в вашей среде.

### Обновление скачалось, но не ставится
Проверьте:
- не заблокирован ли файл антивирусом;
- есть ли права на запись рядом с приложением;
- соответствует ли релиз ожидаемому формату;
- открывается ли страница релиза вручную.
